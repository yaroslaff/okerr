In last task, okerrclient executed set of standard checks and updated (created) indicators. You can manage this set(s) and use different sets of checks on different servers.

Create configuration template 'testconf' (or any other name starting with 'test') at servers page . Colon character not allowed in configuration name. Include any checks in your configuration template (for example: uptime and version). Then, create configuration for your server (with name of server) to link your server to this template. You can know it's name by command hostname -s or set it by argument --name when running okerrclient or option name in configuration file. Server name (hostname) is one word (no dots). If full server name (fqdn) is myserver.example.com, default okerr server name is myserver.

Run okerrclient, it will perform only checks from template you created and assigned to server.
