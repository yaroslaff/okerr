In this lesson we will track log file on linux server and alert if it will grow abnormally fast.

  • Enable okerrmod check linecount like in previous lesson.
  • Decide, which file we will track. For example we can watch apache error log file: /var/log/apache2/error.log. Or you can create special test file, e.g. /tmp/my.log
  • Edit check configuration /etc/okerr/mods-env/linecount and set that log file in PATHLIST
  • Run okerrmod so it would upload data to server
  • On indicator page in okerr UI set value for diffmax as you want. On next update Okerr will alert if file has more new lines then diffmax value. Low values could be used to pass test faster, e.g. 2. Save new indicator configuration.
  • Wait until log files will group enough. If you use test file, you can run date >> /tmp/my.log few times.
  • Run okerrmod again to upload new data to server. Ensure that diffmax limit worked and indicator switched status to 'ERR'.
