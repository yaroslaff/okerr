Для okerr существует уже готовый набор безопасных и удобных проверок сервера. Чтобы удобнее ими пользоваться, установим файл конфигурации для okerrclient командой: cp /usr/local/okerrclient/conf/okerrclient.conf /etc. Затем, отредактируйте файл /etc/okerrclient.conf, указав идентификатор проекта: textid={{textid}} Запустите okerrclient без параметров, чтобы загрузить в okerr состояние всех индикаторов: 

root@braconnier:~# okerrclient 
okerr updated braconnier:uptime = 362949
okerr updated braconnier:la = 0.31
okerr updated braconnier:df-/ = 53.9
okerr updated braconnier:maxlogsz = 997513
okerr updated braconnier:opentcp = 22(sshd) 25(master) 80(apache2) 139(sm..
okerr updated braconnier:version = OK
okerr updated braconnier:df-inodes-/ = 7
okerr updated braconnier:okerr_errors = 0
...
Задача будет засчитана, когда в проекте будет хотя бы один индикатор, установленный таким образом.

Бонус

Будет удобнее если Вы сразу настроите периодическое обновление через cron или как системный сервис (подробности - по ссылке ниже). Чтоб два раза не вставать. Если этого не сделать, то через некоторое время индикаторы "протухнут", перейдут из состояния OK в ERR, о чем вам придет оповещение (alert) на почту и в телеграм.
