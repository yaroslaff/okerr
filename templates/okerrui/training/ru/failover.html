Настройте простую схему аварийного переключения (failover).

В настройках проекта, в разделе Отказоустойчивость (Динамический DNS) добавьте запись с любым DNS именем (например, www). Вам потребуется учетная запись на любом поддерживаемом сервисе dynamic dns, но вы можете использовать тестовый аккаунт "Cloudflare (okerr account for testing)" для того чтобы потренироваться и пройти тренинг, не регистрируясь на других сервисах.

На странице настройки failover, добавьте два индикатора с разными IP адресами и разным приоритетом. Индикаторы не должны быть в режиме настройки. Дождитесь установки нового IP адреса в DNS. Переведите индикатор с наивысшим приоритетом в ERR (можно принудительно сделать это кнопкой 'ERR') и дождитесь когда в DNS серверах установится имя одного из резервных серверов.

Для настоящего использования failover механизма, вам нужно иметь как минимум два вебсервера на разных IP адресах, каждый из которых может обслуживать этот сайт (Для вебсервера Apache - он должен быть указан в ServerName или ServerAlias). Для прохождения тренинга этого не требуется, просто используйте два любых IP адреса, например 1.1.1.1 и 1.1.1.2.
